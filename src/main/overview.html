The Mapotempo Fleet Java Client.


Description :

The Mapotempo-Fleet-Java-Client is designed to synchronised create and update Mapotempo's Missions beetween a server database and a local storage. The main problem solved by mapotempo-fleet-java-client is the offline managing datas. With this library you can access at mapotempo data into a java application even havn't an available connection. You can also create, update or delete data offline.

    What provide mapotempo-fleet-java-client :
  • {@link com.mapotempo.fleet.api.model.CompanyInterface}
  • {@link com.mapotempo.fleet.api.model.UserInterface}
  • {@link com.mapotempo.fleet.api.model.MissionInterface}
  • {@link com.mapotempo.fleet.api.model.MissionStatusTypeInterface}

First connection :

A connection is represented by {@link com.mapotempo.fleet.api.MapotempoFleetManagerInterface}. You can establish a connection with the {@link com.mapotempo.fleet.api.ManagerFactory}, this class provide methods to try to get a {@link com.mapotempo.fleet.api.MapotempoFleetManagerInterface}. Instance will be return into the {@link com.mapotempo.fleet.api.MapotempoFleetManagerInterface.ConnectionVerifyListener callback provide in the method. The first connection necessity an available connection with the Mapotempo server. This is necessary to synchronise {@link com.mapotempo.fleet.api.model.UserInterface} and {@link com.mapotempo.fleet.api.model.CompanyInterface} models. 


MapotempoFleetManagerInterface mMapotempoFleetManager = null;
ManagerFactory.getManager(new JavaContext(), "my_user_name", "my_user_password", new MapotempoFleetManagerInterface.ConnectionVerifyListener {
   {@literal @}Override
    void onServerConnectionVerify(ConnectionVerifyStatus status, MapotempoFleetManagerInterface mapotempoFleetManager);
        mapotempoFleetManager = mapotempoFleetManager;
    }
});

User and Company :

The {@link com.mapotempo.fleet.api.model.UserInterface} and {@link com.mapotempo.fleet.api.model.CompanyInterface} are return respectively by the {@link com.mapotempo.fleet.api.MapotempoFleetManagerInterface#getUser} and {@link com.mapotempo.fleet.api.MapotempoFleetManagerInterface#getCompany} method.

Mission :

To access at {@link com.mapotempo.fleet.api.model.MissionInterface} you need to get a {@link com.mapotempo.fleet.api.model.accessor.MissionAccessInterface}. {@link com.mapotempo.fleet.api.MapotempoFleetManagerInterface} return it using the {@link com.mapotempo.fleet.api.MapotempoFleetManagerInterface#getMissionAccess} method.

MissionStatusType :

To access at {@link com.mapotempo.fleet.api.model.MissionStatusTypeInterface} you need to get a {@link com.mapotempo.fleet.api.model.accessor.MissionStatusTypeAccessInterface}. {@link com.mapotempo.fleet.api.MapotempoFleetManagerInterface} return it using the {@link com.mapotempo.fleet.api.MapotempoFleetManagerInterface#getMissionStatusTypeAccessInterface} method.

Synchronisation :

When the library detecte an available connection and the synchronisation is turn on, it will be synchronise his local database with the mapotempo server database automatically.
Synchronise actions can fail in several case :

  • channel autorisation
  • creating, updating or deleting permission on the model object
